SCREEN 01 V2 HYBRID

Tune In · 주파수 입력으로 방 입장

반원 게이지 라디오 메타포. 4자리 입력하면 인디케이터가 한 칸씩 우측으로 이동해 정중앙 0°에서 LED와 정렬 → "TUNED IN".

  • 반원 게이지 + 9 tick 균등 분포
  • 한 자리 입력 = 22.5° 회전
  • LED 호흡 펄스 + 인디케이터 micro-snap
  • TUNE IN 버튼만 TE Orange (유일한 액센트)

  • NOW만 셔터 활성, 과거는 비활성

03 ·

핵심 결정 사항

absolute rules
RULE 01 LOCKED
3:4 비율 절대 고정
Feed 모자이크의 모든 셀, 인앱 카메라 결과물, Post Detail, NO SIGNAL 박스까지 모두 3:4 비율. 비대칭 모자이크는 큰 셀=2x2 sub-cell 강제 제약 때문에 회피하고 균등 그리드로 단순화.
.cell { aspect-ratio: 3 / 4; }
RULE 02 LOCKED
한 라운드 = 한 송출 · 24H 휘발
2시간 라운드, 한 명당 한 라운드에 사진 1장 또는 영상 1개. 24시간 지난 라운드부터 자동 삭제. "흔적이 쌓이지 않는 방"이 정체성.
round: 2H · ttl: 24H · max video: 5s
RULE 03
인앱 촬영 only · 갤러리 import 없음
사진첩에서 가져올 수 없음. 오직 앱 안에서 실시간으로 찍은 데이터만 송출 가능. 카메라 셔터 좌측을 비워서 시각적으로 신호 (갤러리 진입점 자체 부재).
camera input: live capture only
RULE 04
통합 셔터 · 짧게 사진 / 길게 영상
PHOTO/VIDEO 모드 토글 없음. 단일 셔터: 250ms 이내 release = 사진, 250ms 이상 hold = 영상 시작. 한 손가락만으로 전체 촬영 흐름.
tap < 250ms → photo · hold ≥ 250ms → video
RULE 05
컬러 액센트 단일점 · TE Orange
화면 내 컬러 액센트는 항상 한 곳에만. Tune In = TUNE IN 버튼 / Feed = 셔터 / Camera = 셔터 + USE 버튼. Signal Green은 LIVE/OK 신호로만 점적 사용.
accent fill: TE Orange #F15A24 · 화면당 1개
RULE 06
메타포 = 구조에만, UI = 정밀
무전기 + 필름카메라는 구조와 용어 (Channel, Tune In, Frame, Round). UI는 메타포에 휘둘리지 않고 iPod Nano급 손끝 디테일 정밀도. 메타포 강요로 디자인 어색해지면 메타포가 후순위.
metaphor: structure only · UI: precision first
